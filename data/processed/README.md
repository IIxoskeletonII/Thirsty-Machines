# data/processed

Parquet snapshots produced by `notebooks/01_ingestion.ipynb` (Phase 1,
bronze layer). Each parquet is a schema-faithful materialisation of a
raw input under `data/raw/<source>/`, with no transforms beyond what
`pd.read_csv` / `pd.read_excel` does natively. Documented preprocessing
(AWS skiprows, Ember column reconciliation, IM3 layer filtering,
Aqueduct sentinel handling, indicator/weight filters, etc.) is applied
in Phase 2.

This directory is gitignored and regenerated by re-running the ingestion
notebook on a fresh kernel.

## Sources

- **epoch/** (1 parquet) — Epoch AI Notable AI Models. Per-model
  metadata: organization, publication date, parameters, training
  compute, hardware, training power draw.
- **ember/** (2 parquet) — Ember electricity data in long format.
  Yearly country-level (`Area` + `ISO 3 code`) and US state-level
  (`State` + `State code`) measures across many `Variable`s
  including `CO2 intensity`.
- **im3/** (1 parquet) — IM3 open-source data center atlas. Per-
  facility records (operator, name, location, square footage,
  layer type) for US data centers.
- **aqueduct/** (2 parquet) — WRI Aqueduct 4.0 water-risk rankings.
  Country-level baseline (`country_baseline`) and province-level
  baseline (`province_baseline`) sheets, scored across multiple
  indicators and weighting schemes.
- **jegham/** (5 parquet) — Jegham et al. LLM inference footprint.
  Three environmental-footprint files (frozen `DataSnapshotOct26`,
  live `artificialanalysis_environmental`, monthly time series) and
  two hyperscaler infrastructure-multiplier files (AWS, Microsoft
  region-level PUE/WUE/CIF).

Total: 11 parquet files across 5 source subdirectories.
