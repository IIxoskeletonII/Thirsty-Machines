# data/processed/tableau

Seven CSV extracts produced by `notebooks/04_tableau_extracts.ipynb`
for consumption by the six Tableau workbooks under `tableau/` (plus
`c8_flow.csv`, which feeds the Component 8 system-map static). Files
in this directory are committed — unlike the parquets in sibling
processed-data directories, which are gitignored and regenerated by
notebook re-run — so a fresh clone can open the Tableau workbooks
without running Python first.

The extracts are regenerable: rerun `notebooks/04_tableau_extracts.ipynb`
on a fresh kernel against the silver-layer parquets in
`data/processed/integrated/`. The notebook applies three small
derivations the silver layer defers to this presentation layer
(Component 5's 100B-energy correction across three MWh-energy columns,
Component 2's `is_unmatched` flag for 2023-vintage models without
long-prompt benchmarks, and Component 8's flow-shape reshape into
Sankey nodes and edges).

Files:

- `c2_model_energy.csv` — Component 2 ranked dot plot (66 models × 3
  query-length tiers, 198 rows).
- `c3_regions.csv` — Component 3 global cloud-region map (37 rows
  verbatim from `regions_geographic.parquet`).
- `c4_trajectory.csv` — Component 4 training compute trajectory (the
  15 frontier models with disclosed FLOP).
- `c5_aggregate_scaling.csv` — Component 5 aggregate scaling, long
  format with the published-vs-derived correction applied to the three
  MWh-energy 100B-scale columns.
- `c6_small_multiples.csv` — Component 6 efficiency-vs-demand small
  multiples (12 selected models × 3 tiers, 36 rows).
- `c8_flow.csv` — Component 8 Sankey flow trace (5 edges, GPT-4o
  (Aug) long → Virginia path).
- `region_coordinates.csv` lives at `data/reference/region_coordinates.csv`
  rather than in this directory; it is a hand-curated reference table
  of the 37 cloud-region coordinates and is referenced by Tableau as
  a join companion to `c3_regions.csv`.
